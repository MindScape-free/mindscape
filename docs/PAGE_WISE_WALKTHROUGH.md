# 🧠 MindScape — Page-by-Page Walkthrough

A comprehensive tour of every screen and workflow in MindScape, designed for new users.

---

# 📋 Overview

MindScape is an **AI-powered knowledge visualization platform** that transforms unstructured information (PDFs, videos, websites, text) into **interactive, explorable mind maps** and **knowledge graphs**. It combines deterministic structural extraction with AI synthesis to produce accurate, hallucination-resistant visual knowledge representations.

---

# PAGE 1: Home / Landing

## 🎯 Purpose

The entry point where users start their knowledge journey. The hero page is designed as a powerful input hub.

## 🖼️ What the User Sees

### Hero Section — The Command Center

- **Rotating headlines** — dynamic taglines rotate every 60s ("Turn complex data into structured knowledge", "Synthesize chaos into visual clarity")
- **Visual Intelligence Engine badge** — brand positioning
- **Source input bar** — the primary action point:
  - Type any **topic, URL, or YouTube link**
  - Attach **PDF, image, or text files**
  - Paste website URLs (auto-scraped in background)
- **Mode selector** — three tabs: `Single`, `Compare`, `Multi-Source`
- **Configuration controls**:
  - **Depth**: Quick → Balanced → Detailed → Auto (AI decides)
  - **Persona**: Teacher, Concise, Creative, Cognitive Sage
  - **Language**: Full language selector (50+ languages)

### How "Single Mode" Works

1. User enters a topic or uploads a file
2. If it's a **YouTube URL** → redirects to canvas with transcript extraction
3. If it's a **website URL** → redirects to canvas with web scraping
4. If it's a **file** (PDF/image/text) → session is created with file content
5. If it's **plain text** → navigates to canvas with topic as query parameter
6. **Auto-depth detection** — the system analyzes topic complexity using keyword scoring (technical, academic, scientific keywords) to choose the right depth

### How "Compare Mode" Works

- Two input fields for side-by-side topic analysis
- Supports files for each topic independently
- Generates a unified comparison with shared/different dimensions

### How "Multi-Source Mode" Works

- Drag-and-drop multiple sources (PDFs, URLs, text snippets)
- All sources are merged into a single unified context
- Generates a comprehensive cross-referenced mind map

### Feature Sections Below the Fold

- **About MindScape** — brand story
- **What MindScape Does** — feature grid (Knowledge Graphs, Compare Mode, Deterministic Structuring, Multi-Depth, Share & Collaborate, Multi-Source)
- **How It Works** — 4-step pipeline (Input Processing → Knowledge Engine → AI Synthesis → Visual Rendering)

### Background Interactions

- **Floating chat button** (bottom-right) — opens the AI chat assistant
- **Scroll-to-top button** (appears after scrolling 400px)
- **Onboarding Wizard** — automatically triggers for first-time users (4-step walkthrough)
- **Navbar** — Home, Community, Library, profile/auth, pollen balance, XP rank badge

# PAGE 2: Canvas / Mind Map View

## 🎯 Purpose

The core mind map exploration workspace where users interact with their generated knowledge.

## 🖼️ Route: `/canvas?topic=X` or `/canvas?sessionId=X` or `/canvas?mapId=X`

### Mind Map Rendering

- **Hierarchical Accordion View** (default) — expandable/collapsible tree:
  - **Topic** → **SubTopics** → **Categories** → **SubCategories**
  - Each node has icons, names, and insights
  - Depth indicator badge
- **Map View** (full-screen portal) — interactive radial/network graph via `MindMapTreeView`
- **Compare View** — side-by-side comparison with shared nexus, dimensions, and synthesis horizon

### Toolbar — Top of Map

| Feature | Description |
|---|---|
| **Save** | Persists map to Supabase |
| **Duplicate** | Creates a copy in your library |
| **Translate** | AI-powered translation to any language |
| **Regenerate** | Re-runs the AI generation |
| **Publish** | Makes map public in Community Dashboard |
| **Share** | Generates unlisted share link |
| **Summary** | Opens AI-generated text summary dialog |
| **Nested Maps** | Opens dialog showing all sub-maps (expansions) |
| **Gallery** | Opens image gallery of generated visuals |
| **AI Content** | Opens all AI-generated content in one dialog |
| **Global Quiz** | Generates a quiz for the entire topic |

### Per-Node Interactions

Clicking any **SubCategory** node opens an **Explanation Dialog** with:

- **Explanation levels**: Beginner → Intermediate → Expert
- **Examples**: Real-world analogies generated by AI
- **Confidence rating**: User can rate explanation quality
- **Micro-quiz**: Test understanding of this specific concept
- **Enrichment**: Auto-fetched additional context (fire-and-forget)
- **Visual Insight Lab**: Generate AI images for this concept
- **Nested Expansion**: Create a sub-map for deeper exploration
- **Practice Questions**: Related questions for study

### Automated Background Actions

- **Auto-summarization**: When map is fully loaded, AI generates a summary
- **Auto-save**: Changes (images, explanations, expansions) are debounced and saved
- **Automatic language sync**: UI tracks current translation state

### Synthesis / Knowledge Alchemy Mode

When toggled from the toolbar:

1. A floating control panel appears at the bottom-right
2. Click any two nodes in the map to select them
3. Click "FUSE KNOWLEDGE"
4. AI generates a hybrid concept combining both nodes
5. The result is added as a new node to the map

# PAGE 3: Chat Panel (Slide-over)

## 🎯 Purpose

An AI assistant that lives alongside the mind map for Q&A, exploration, and quizzes.

## 🖼️ Opens as a sliding sheet from the right

### Chat Interface

- **Message history** with user/AI bubbles
- **Streaming responses** — text appears in real-time with cursor indicator
- **Thought trace** — shows AI reasoning steps (expandable)
- **Tool calls** — displays when AI uses tools (search, etc.) with status indicators
- **Markdown rendering** — supports code blocks, tables, lists, math (KaTeX), and more

### Input Area

- Text input with auto-resize
- **File attachments**: PDF, images, text files
- **URL auto-scraping**: Pasting a URL auto-fetches and attaches content
- **Document context toggle**: Switch between "Standard" and "Aware" mode (uses source file as context)
- **Voice input**: Speech-to-text via Web Speech API
- **Persona selector**: Teacher, Concise, Creative, Sage

### Suggested Actions (empty state)

- Deep Dive, Quick Quiz, Key Concepts, Fact Check
- Suggestion prompts shuffled for variety

### Quiz System

- **Difficulty selector**: Easy / Medium / Hard
- **AI-generated quizzes** with concept tagging
- **Adaptive results**: Shows correct/wrong, strong/weak areas
- **Adaptive deepening**: Low-scoring areas trigger AI generation of new map nodes
- **Quiz result card**: Score breakdown, per-tag performance

### Session Management

- **History view**: Past chat sessions
- **Pinned messages**: Star important Q&As (syncs to mind map)
- **Export to PDF**: Download entire chat as a formatted PDF
- **Copy / Regenerate**: Per-message actions

### Related Questions

- AI automatically generates 3-5 follow-up questions after each response
- Click any to send as a new message
- Collapsible section

### Create Mind Map from Chat

- Each AI response has a 🧠 button
- Click to generate a mind map from the response content
- The generated map appears in the canvas

# 🧠 Knowledge Flow — How It All Connects

```
User Input → Knowledge Engine → AI Synthesis → Interactive Map → Exploration & Learning
```

### Step 1: Input Processing

The user provides content via any of these pathways:

- **Text topic** — directly submitted
- **PDF** — parsed via pdf.js with progress tracking
- **Image** — base64-encoded, optionally resized
- **Website** — content extracted via web scraper
- **YouTube** — transcript extracted via youtube-dl
- **Multi-source** — all the above merged into unified context

### Step 2: Knowledge Engine (SKEE)

Before AI touches the data, MindScape's **Structural Knowledge Extraction Engine** runs deterministically:

- **Heading Detector** — identifies document structure
- **Section Splitter** — breaks content into logical sections
- **Concept Extractor** — identifies key concepts and terms
- **Relationship Detector** — maps relationships between concepts
- **Keyword Extractor** — pulls meaningful keywords
- **Graph Builder** — builds an initial structural graph

This ensures **zero hallucinated headings** and **structural accuracy** before AI synthesis.

### Step 3: AI Synthesis

The structured content plus user configuration is sent to:

- **Pollinations.ai** (sole provider, with OpenRouter fallback planned in future)
- Multiple AI flows depending on source type:
  - `generate-mind-map` — general topic-to-map
  - `generate-mind-map-from-pdf` — PDF-specific
  - `generate-mind-map-from-image` — vision analysis
  - `generate-mind-map-from-website` — web content
  - `youtube-mindmap` — video transcripts
  - `generate-comparison-map` — compare mode
- **Streaming generation** for chat responses
- **Schema-based validation** — AI output is validated against Zod schemas

### Step 4: Post-Processing & Sanitization

Raw AI output goes through `mapToMindMapData()` which:

- Extracts only known fields (no raw spread — security measure)
- Validates and normalizes all data
- Assigns stable IDs to every node
- Computes node counts in a single pass
- Strips hallucinated or unexpected properties

### Step 5: Interactive Visualization

The sanitized data is rendered as:

- **Accordion tree** (default, best for exploration)
- **Radial graph** (best for overview)
- **Compare view** (side-by-side with shared nexus)

### Step 6: Continuous Learning Loop

Users don't just view — they **interact**:

- Click nodes → get explanations at 3 levels
- Take quizzes → AI identifies weak areas → generates new nodes
- Generate images → visualize concepts
- Expand sub-maps → infinite depth exploration
- Chat with AI → ask follow-ups, debate concepts
- Pin responses → save insights for later

# 🔧 Stack & Architecture

```
Frontend:  Next.js 16 (App Router) + React 18 + TypeScript + Tailwind CSS
UI:        Framer Motion + Radix UI + Lucide Icons + shadcn/ui
Backend:   Next.js Server Actions + Supabase (Postgres + Auth)
AI:        Pollinations.ai (sole provider)
           Models: Flux (images), Qwen (text), Gemini Search
Storage:   Supabase DB + Session Storage (client cache)
Auth:      Supabase Auth (email/password + Google OAuth)
Rendering: pdf.js (PDFs), KaTeX (math), Prism.js (code)
State:     React Contexts + Custom Hooks + Local Storage
```
